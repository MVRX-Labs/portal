If doing things with long-running operations, use Trigger.dev and check out the TRIGGER_DETAILS.md file for more information.

For any long running jobs, remember to send slack notifications on failure

Where possible, reduce coupling. Eg the calendar sync must operate independently of the linkedin audit.

Do not expect accounts/contacts to always be accurate.

Every time you notice that you made a mistake, or went down a wrong path, write a note in the NOTES.md file.
